Chapitre 5

    1. Qu`est ce que Git et a quoi il sert ? :

      2. Git est un logiciel de versioning qui permet de sauvegarder les changements effectuées sur un projet ce qui facilite la gestion de ces dernier car il rend la coordination du travail entre plusieurs personne possible et simple .

    2. Les deux modèle des logiciels de gestion de version :

      3. Le modèle de gestion sont soit construit sur le modèle centralisé qui consiste à la centralisation du code source et les différents utilisateur doivent se connecter sur ce serveur , ou bien le modèle de distribué où chaque utilisateur doit télécharger et héberger l`intégralité du code sur sa propre machine . Git utilise le modèle distribué car c`est simple , manipulable et plus sécurisé .

    3. Utiliser Git : ligne de commande, console et interface graphique et son installation :

      4. on utilise la console ( langage de commande) pour installer et utiliser Git et pour l`installer la manière la plus simple est télécharger la dernière version sur ce site : http://git-scm.com/downloads

    4. Paramétrage de Git :

      5. Pour paramétrer Git, tu dois configurer ton nom d`utilisateur et ton adresse e-mail en utilisant les commandes « git config –global user.name 'ton nom d`utilisateur' » et « git config –global user.e-mail 'ton adresse e-mail' »

    5. Démarrer un dépôt Git :

      6. Un dépôt reflète la copie d`un fichier. On peut crée un dépôt Git en le clonant d`un autre dépôt ou en important un répertoire déjà existant

    6. La gestion des informations selon Git :

      7. Git considère différemment les donner où il pense à la manière d`un flux instantané , chaque instantané est stocké dans une base de donné locale .

    7. Les états des fichiers :

      8. Un fichier peut avoir deux grands états dans Git : suivie, non suivie. Si il est enregistré en base alors le fichier est suivie et celui qui n`as pas été indexé n`est pas suivie. Les fichiers suivie peuvent avoir trois états : modifié, indexé , validé .

    8. Les zones de travail :

      9. Les états des fichiers sont liées à des zones de travail dans Git en fonction de son état dont tout projet Git est composé de trois sections : le répertoire de travail, la zone d`index et le répertoire Git.

    9. Créer un dépôt Git d`un répertoire existant

      10. Pour créer un dépôt Git à partir d`un répertoire existant, tu dois exécuter la commande « Git int » dans le terminal du répertoire. Cela initialisera un dépôt Git dans le répertoire et tu pourra commencer à ajouter des fichiers et des commits. Pour avoir des informations sur l`état des fichier on utilise « Git statuts » et pour les indexer on utilise « Git add » et enfin pour les valide et les ajouter en base la commande « La seconde manière de démarrer un dépôt Git est de le cloner grâce à la commande « git commit » est idéal.

    10. Cloner un dépôt Git :

      11. La seconde manière de démarrer un dépôt Git est de le cloner grâce à la commande « Git clone » , une fois finis nous pourrons transmettre nos fichier.

    11. Ajouter, modifier, actualiser notre projet, consulter l`historique ou annuler les modifications :

      12. Grâce à notre logiciel de gestion de version l`ajout et la modification d`un fichier est possible, pour cela on va devoir utuliser les commande « git add » et git commit » une fois faite la manière la plus simple de consulter l`historique de nos modification est d`utiliser la commande « Git.log » et en cas de regret on pourra annuler les modifications avec la commande « git checkout-nom-du-fichier » ou « Git restore ».

    12. Qu`est ce qu`une branche ? :

      13. Créer une branche c`est créer une copie du projet pour le développer Git a une approche facile pour crée une branche il suffit simplement de créer un nouveau pointeur.
